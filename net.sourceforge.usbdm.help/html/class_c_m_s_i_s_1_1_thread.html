Detailed Description

Wrapper for CMSIS Thread.

//
// Thread example
//
void threadExample() {
static auto threadFn = [] (const void *) {
for(;;) {
BLUE_LED::toggle();
osDelay(2000);
}
};
static Thread thread(threadFn);
BLUE_LED::setOutput();
thread.run();
printf(" thread::getId() = %p\n\r", thread.getId());
}
Examples:
cmsis-cpp-mailQueue.cpp, cmsis-cpp-messageQueue.cpp, cmsis-cpp-thread-mma845x.cpp, and cmsis-cpp-thread.cpp.

static osEvent CMSIS::Thread::signalWait ( int32_t  signals,

Wait for one or more Signal Flags to become signaled for the current thread.

Parameters
[in]signalsWait until all specified signal flags set or 0 for any single signal flag.
[in]millisecHow long to wait in milliseconds. Use osWaitForever for indefinite wait
Returns
Structure describing event occurring with status:
- osOK: no signal received when the timeout value millisec was 0.
- osEventTimeout: signal not occurred within timeout
- osEventSignal: signal occurred, value.signals contains the signal flags; these signal flags are cleared.
- osErrorValue: the value signals is outside of the permitted range.
- osErrorISR: osSignalWait cannot be called from interrupt service routines.

static osStatus CMSIS::Thread::yield ( )
inlinestatic

Pass control to the next thread that is in state READY.

If there is no other thread in the state READY, the current thread continues execution and no thread switching occurs.

Returns
osOK: The function has been correctly executed.
