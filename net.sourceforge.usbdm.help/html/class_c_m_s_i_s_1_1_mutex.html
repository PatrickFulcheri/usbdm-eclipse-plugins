Detailed Description

Wrapper for CMSIS mutex.

A Mutex can be used to:

  • Synchronize the execution of threads
  • Protect access to shared resources. (Implement critical region protection).
Note
Mutex cannot be used in an ISR
Examples:
cmsis-cpp-thread-mma845x.cpp, and cmsis-cpp-timer-mma845x.cpp.

◆ release()

osStatus CMSIS::Mutex::release ( )
inline

Release mutex.

Release the mutex that was obtained with wait().
Other threads that are currently waiting for this mutex will be put into the READY state.

Returns
osOK: The mutex has been correctly released.
osErrorResource: The mutex was not obtained beforehand.
osErrorISR: osMutexRelease cannot be called from interrupt service routines.

◆ wait()

osStatus CMSIS::Mutex::wait ( uint32_t  millisec = osWaitForever)
inline

Obtain mutex.

This method may wait until the mutex is available.
The millisec parameter allow the wait time to be specified.

Parameters
[in]millisecHow long to wait in milliseconds.
Use osWaitForever for an indefinite wait
Use 0 to immediately fail if the mutex is not available.
Returns
osOK: The mutex has been obtained.
osErrorTimeoutResource: The mutex could not be obtained in the given time.
osErrorResource: The mutex could not be obtained when no timeout was specified.
osErrorParameter: The parameter mutex_id is incorrect.
osErrorISR: osMutexWait cannot be called from interrupt service routines.
